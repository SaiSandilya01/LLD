# Object-Oriented Programming (OOP) Pillars

This directory contains simple, foundational examples of the four pillars of Object-Oriented design:

1. **Encapsulation** (`Encapsulation.java`): Bundling data and methods that operate on that data into a single unit, hiding the internal state, and requiring all interaction to be performed through an object's methods.
2. **Abstraction** (`Abstraction.java`): Hiding complex implementation details and exposing only the essential features of an object.
3. **Inheritance** (`Inheritance.java`): A mechanism where you can derive a class from another class for a hierarchy of classes that share a set of attributes and methods.
4. **Polymorphism** (`Polymorphism.java`): The ability of different objects to respond in a unique way to the same method call, typically achieved through interfaces or abstract classes.
